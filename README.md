# Infosys-Document-Workbench 1.3.0.0

Streamline document handling with Infosys Document Workbench. Tackle the diversity of formats effortlessly, blending low-code solutions for semi-structured content and GenAI RAG for the unstructured. Simplify your document processing, smartly and efficiently.

# **How does it Work**

1. Organizations receive large volume of **unstructured documents** from their customers/ partners/ employees etc.
2. These documents are received in different formats such as **scanned images/pdf/word documents** etc.
3. They are manually **reviewed/validated**, and data is entered into backend transaction management systems by Operations Agents.

While organizations try RPA tools/AI tools for intelligent automation of such processes, it is difficult to infuse these technologies in legacy systems since they do not support AI/ Automation natively.

As a result, **business does not realize values from such implementations**. Thus, there is need to create unified UI which is able to bring enterprise data, insight and action together at the point of handling business transactions.

**Infosys Document workbench** is the solution to this problem. It combines **intelligence from text analytics and Generative AI** with **action using Robotic Process Automation (RPA) platform & OCR capabilities** to automate the steps required for attribute extraction prior to triggering of downstream business processing. It also offers a **Unified UI** that leverages AI/Automation techniques to improve efficiency and effectiveness by providing **Assisted Automation** to the process of handling of documents.

There are broadly **two approaches** used for attribute extraction.


1. For **semi-structured documents**, it uses a **low-code no-code approach** where the information to be extracted is treated as a **pattern** and configured as a **region definition** (a JSON structure) which makes it a **template agnostic** solution.
2. For **unstructured and large documents**, it uses **GenAI** in the form of **RAG-based LLMs** which allows for **semantic search and Q&A** to extract information.
